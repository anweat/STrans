# M07：认证与权限 — AuthStore + SQLite 会话

## 1. 模块定位

认证主链路位于 `backend/app/services/auth_store.py`，由三个子组件构成：

| 组件 | 文件 | 职责 |
|---|---|---|
| `AuthStore` | `auth_store.py` | 密码哈希、用户 CRUD、持久化会话、验证码、审计 |
| `auth_store`（单例） | `auth_store.py` | 全局认证服务实例 |
| `current_user/current_admin` | `main.py` | Bearer Token 校验与 RBAC 依赖注入 |

`backend/app/services/auth.py` 只保留兼容导出，避免旧导入路径形成第二套认证实现；结题行为与安全口径均以 `auth_store.py` 为准。

## 2. 认证方案

### 2.1 密码安全

```python
def _hash_password(self, password, salt=None):
    salt = salt or secrets.token_hex(16)         # 16字节随机盐
    digest = hashlib.pbkdf2_hmac(
        "sha256",
        password.encode("utf-8"),
        salt.encode("ascii"),
        120_000                                    # 12万次迭代
    )
    return f"pbkdf2_sha256${salt}${base64(digest)}"
```

**安全属性**：
- PBKDF2-SHA256，12 万次迭代 → 暴力破解计算成本高
- 16 字节随机盐 → 相同密码不同哈希
- 首次建库必须通过 `STRANS_ADMIN_PASSWORD` 提供至少 12 位的管理员初始密码，仓库不再内置默认口令

### 2.2 会话 Token

```python
def create_session(self, user_id):
    token = secrets.token_urlsafe(32)
    expires_at = now + timedelta(hours=12)
    INSERT INTO user_sessions(token, user_id, created_at, expires_at)
    return token
```

**设计**：
- Token 使用密码学安全随机数生成，不包含用户信息；
- 每次鉴权都联查 `user_sessions` 与启用状态，默认 12 小时过期；
- 登出、改密、禁用用户和管理员重置密码都会删除相应会话。

## 3. 用户管理

### 3.1 数据库 Schema

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    role TEXT DEFAULT 'user',    -- 'admin' / 'user'
    enabled INTEGER DEFAULT 1,
    created_at TEXT NOT NULL,
    last_login_at TEXT
);

CREATE TABLE user_sessions (
    token TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    created_at TEXT NOT NULL,
    expires_at TEXT NOT NULL
);

CREATE TABLE audit_logs (
    id INTEGER PRIMARY KEY,
    username TEXT,
    action TEXT,                 -- 'create_camera' / 'delete_whitelist' / ...
    target TEXT,                 -- 操作对象标识
    detail TEXT,                 -- 详细描述
    created_at TEXT
);
```

### 3.2 RBAC 权限模型

| 角色 | 权限范围 |
|---|---|
| admin | 全部 API + 用户管理 + 摄像头管理 + 白名单修改 + 模型配置 |
| user | 查看监控、查询历史、查看告警、修改自己密码、查看白名单 |

### 3.3 API 权限注入 (main.py)

```python
def current_user(authorization=Header(None)):
    user = auth_store.get_user_by_token(token)
    if not user: raise HTTPException(401)
    return user

def current_admin(user=Depends(current_user)):
    if user["role"] != "admin": raise HTTPException(403)
    return user

# 使用:
@app.get("/api/admin/users")             # → Depends(current_admin)
@app.get("/api/history")                  # → Depends(current_user)
@app.get("/api/health")                   # → 无认证
```

## 4. 审计日志

所有管理操作均记录：

```python
auth_store.add_audit(username, action, target, detail)
```

审计覆盖的操作类型：
- `create_camera` / `update_camera` / `delete_camera`
- `upsert_whitelist` / `delete_whitelist`
- `update_user` / `delete_user` / `reset_password`
- `change_password`
- `configure_scheduler`
- `update_model_config`
- `delete_history` / `purge_history`
- `handle_incident` / `export_evidence`
- `generate_report`

## 5. API 接口

### 认证接口

| 端点 | 方法 | 权限 | 说明 |
|---|---|---|---|
| `/api/auth/captcha` | GET | 公开 | 获取验证码(ID+图片) |
| `/api/auth/register` | POST | 公开 | 注册(需验证码) |
| `/api/auth/login` | POST | 公开 | 登录(需验证码) → token |
| `/api/auth/me` | GET | user | 当前用户信息 |
| `/api/auth/logout` | POST | user | 登出(销毁token) |
| `/api/auth/password` | PUT | user | 修改自己密码 |

### 管理接口

| 端点 | 方法 | 权限 | 说明 |
|---|---|---|---|
| `/api/admin/users` | GET | admin | 用户列表 |
| `/api/admin/users/{id}` | PUT | admin | 修改角色/启停 |
| `/api/admin/users/{id}/password` | PUT | admin | 重置密码 |
| `/api/admin/users/{id}` | DELETE | admin | 删除用户 |
| `/api/admin/audit` | GET | admin | 审计日志 |

### 安全约束

- 不能禁用/删除自己的账号
- 修改密码后要求重新登录 (`reauth_required: true`)
- 视频源地址在审计中脱敏
- 首次建库缺少合规 `STRANS_ADMIN_PASSWORD` 时拒绝启动
- 登录页不预填密码
